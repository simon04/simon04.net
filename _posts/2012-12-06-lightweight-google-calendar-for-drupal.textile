Today, I needed to integrate a lightweight and configurable calendar in a Drupal 6 page. The calendar entries should be obtained from a (public) Google calendar automatically. I ended up with the Drupal modules <a href="https://drupal.org/project/gcal_events">gcal_events</a> and <a href="https://drupal.org/project/insert_block">insert_block</a>. Here's what I did:
<ol>
<ol>
	<li>Download the two modules, apply the patch from <a href="https://drupal.org/node/373071">373071</a> in order to obtain localized month names, obtained the needed  <a href="http://dev.simplepie.org/SimplePie.compiled.php">simplepie.inc</a> (to go into <code>/modules/gcal_events/simplepie.inc</code>)</li>
	<li>Upload to webserver (as well as create a writeable directory <code>/module/gcal_events/cache</code>)</li>
	<li>Enable the modules in <code>/admin/build/modules</code></li>
	<li>Edit calendar block as needed: <code>/admin/build/block/configure/gcal_events/0</code></li>
	<li>Add <em>insert block filter</em> to a filter of choice (<code>/admin/settings/filters</code>)</li>
	<li>Create a page <code>/node/add/page</code>, insert <code>[block:gcal_events=0]</code> as content (select filter from previous step)</li>
	<li>When using caching, exclude page from previous step using module <a href="https://drupal.org/project/cacheexclude">cacheexclude</a> (<code>/admin/settings/cacheexclude</code>)</li>
	<li>Done :-)</li>
</ol>
</ol>
The result is currently visible as a demo page under <a href="http://martinmallaun.com/kalender">http://martinmallaun.com/kalender</a>.
